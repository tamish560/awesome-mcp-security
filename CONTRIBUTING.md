# Contributing to Awesome MCP Security

Thank you for your interest in contributing. This list aims to be the most comprehensive and well-maintained resource for MCP security tools, research, and best practices.

## What we accept

- Open source MCP security tools (scanners, firewalls, proxies, gateways, redaction tools)
- Commercial MCP security products with a free tier
- Security research papers and standards documents
- Blog posts and guides with substantive technical content
- MCP security testing frameworks and red-team tools

## What we do not accept

- General MCP tools without a security focus
- Closed-source tools without a usable free tier
- Marketing content or listicles without technical depth
- Duplicate entries (search before submitting)

## How to submit

1. Check that your tool is not already listed
2. Open a pull request adding your entry to the appropriate section
3. Use the format: `- [owner/repo](url) - Description. By Author/Org.`
4. Keep descriptions to one or two sentences. No marketing language.
5. Alphabetical order within each section where possible

## Format

```markdown
- [owner/repo](https://github.com/owner/repo) - One or two sentence description. By Author.
```

## Review process

Pull requests are reviewed within 3-5 days. We may ask for changes to descriptions or categorization.
